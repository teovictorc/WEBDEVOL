Como utilizar o assistente

 

O assistente do Sistema WFA Web pode ser acionado de qualquer parte do sistema, clicando sobre o link AJUDA localizado na parte superior direita da tela, ou através do menu AJUDA.

Para obter ajuda sobre um tópico, basta clicar sobre a frase. Sugere-se que este assistente, ao ser lido, seja acompanhado pela navegação no Sistema WFA Web, para um maior entendimento.

Efetuando login no sistema

 

Ao iniciar o sistema, a primeira etapa é efetuarmos login no sistema, conforme tela abaixo:

 

  1. Login: deve ser informado login de acesso ao sistema. O login de acesso será normalmente o código de sua loja junto a Arezzo;
  2. Senha: deve ser informada a senha de acesso ao sistema;
  3. Esqueci minha senha: esta opção deverá ser utilizada quando o usuário não lembrar sua senha de acesso ao sistema;
  4. Cadastrar os Tipos de Recurso existentes na produção;
  5. Cadastrar os Recursos existentes na produção;
  6. Cadastrar as Linhas de produção;
  7. Cadastrar as Operações;
  8. Cadastrar as Características;
  9. Cadastrar as Matérias Primas de Laboratório;
  10. Cadastrar as Unidades de Consumo;
  11. Cadastrar os Pesos da medida.

Gerenciando Fichas Técnicas

 

O gerenciamento de Fichas Técnicas, no sistema PCPFCC, necessita de uma pré-configuração da base de dados, antes que sejam inseridas as fichas técnicas. Para maiores informações, consulte o tópico Preparando a base de dados para inserir uma Ficha Técnica.

O módulo de Fichas Técnicas, é acionado através da opção Pesquisa e Desenvolvimento, do menu principal. Ao ser acionado, aparece uma relação de todas as Fichas Técnicas registradas no sistema, pertencentes ao Tipo de Ficha Técnica que o usuário logado tem direitos de acesso.

As colunas que estão disponíveis neste grid de visualização são:

·         Código FT  código com o qual foi registrada a FT;

·         Nome da Ficha Técnica  nome com o qual a FT foi registrada;

·         Aprovada  indica se a FT é ou não aprovada. Caso não seja aprovada, é considerada experimental;

·         Ativa  indica se a FT é ou não ativa no sistema. Somente para as FT ativas é permitida a impressão de OP;

·         Tipo de FT  tipo de FT associada à FT;

·         Família  família a qual esta FT pertence;

·         Data criação  data que a FT foi criada;

·         Criada por  usuário responsável pela criação da FT.

Neste momento, também estão disponíveis algumas ações, e as mesmas estarão disponíveis em todas as janelas:

  • Incluir  inclui uma nova FT;
  • Alterar  altera a FT selecionada. Também pode ser ativada com um duplo clique uma FT;
  • Excluir  exclui a FT selecionada;
  • Cancelar  cancela a operação em andamento;
  • Imprimir  imprime uma relação de FT;
  • Pesquisar  pesquisa FT por alguns critérios;
  • Copiar Fctec  duplica a FT selecionada;
  • Sair  sai do modulo.

As opções incluir e alterar são operacionalmente idênticas, sendo que a única diferença é que no caso de uma alteração, os campos já vem inicializados com os dados contidos na FT (ou pasta) corrente. Quando se estiver em uma operação de inclusão ou alteração, uma serie de pastas estarão disponíveis, e todas deverão ser preenchidas para que a FT seja considerada completa. Todas as pastas estão vinculadas a FT selecionada na primeira pasta (Ficha Técnica). Serão descritos, portanto, os procedimentos específicos para se criar ou alterar uma FT:

a)      Pasta Ficha Técnica

1.       Informar os seguintes atributos:

                                                              i.      Código da Ficha Técnica  código interno da FT, não é utilizado para fins de comercialização dos produtos a ela vinculados;

                                                            ii.      Nome da Ficha Técnica  nome interno da FT. Da mesma forma que o código, não é utilizado para fins de comercialização;

                                                          iii.      Aprovada  indica se a FT está aprovada ou não. Caso não esteja, é uma FT experimental;

                                                           iv.      Ativa  indica se a FT está disponível para emissão de OP;

                                                             v.      Tipo de Ficha Técnica  tipo de FT a qual a FT está vinculada;

                                                           vi.      Família da Ficha Técnica  família a qual a FT está vinculada;

                                                         vii.      Grade da Ficha Técnica  indica o tipo de grade vinculada a esta FT. Somente solicitará a grade se o Tipo de FT tiver grade;

                                                       viii.      Lote Mínimo de Produção  indica a quantidade mínima a ser produzida para esta FT;

                                                          ix.      Percentual de Perda  indica o percentual de perda aceitável para esta FT;

                                                            x.      Data de Criação  data de criação da FT (criada automaticamente);

                                                          xi.      Criada pelo Usuário  usuário responsável pelo cadastramento da FT (criado automaticamente).

2.       Após inserir os atributos, clicar sobre o botão Gravar, para que os dados inseridos ou alterados sejam gravados no banco de dados;

3.       Editar o arquivo de CEP (Controle Estatístico de Processo). Este arquivo será impresso em toda OP para esta FT. Ao se clicar sobre o botão associado ao CEP, o aplicativo MS Word será acionado, e o arquivo, após editado, deverá ser gravado.

b)      Pasta Versão

Nesta pasta, são cadastradas todas as Versões da FT. Quando selecionada esta pasta, é apresentado um grid de visualização com as seguintes colunas:

·         Versão  número seqüencial da versão da FT. Nesta coluna, ainda pode aparecer um símbolo de cifrão ($), o que indica que esta versão é a Versão de Custo da FT;

·         Nome da versão  nome associado à versão da FT. Não é utilizado para fins de comercialização dos produtos associados à FT;

·         Aprovada  indica se a versão esta ou não aprovada;

·         Ativa  indica se a versão esta ou não ativa;

·         Custo da matéria prima  valor de custo das matérias primas associadas à versão da FT;

·         Custo operacional  custo operacional associado à versão da FT;

·         Data ultima alteração  data em que a versão foi alterada pela ultima vez (gerada automaticamente);

·         Última alteração por  usuário responsável pela ultima alteração (também gerado automaticamente).

1.       Neste momento, também estão disponíveis cinco botões de ações, logo abaixo do grid, quais sejam:

·         Definir versão selecionada como versão de custo - $ à esta ação faz com que a versão selecionada se torne a versão de custo da FT. Somente usuários autorizados podem realizar esta operação;

·         Incluir nova versão - / à abre uma janela para inserção de uma nova versão de FT;

·         Alterar versão selecionada - 1 à abre uma janela para alterar  versão selecionada no grid;

·         Excluir versão selecionada - O à exclui a versão selecionada no grid. Somente permite excluir se não tiver nada vinculado a esta versão;

·         Copiar versão da Fctec - 22 à cria uma versão de FT com base na versão selecionada no grid;

2.       Nas ações incluir e alterar, será aberta uma janela contendo três pastas:

·         Dados da versão: contém os dados básicos da versão da FT;

·         Partes da versão: contém  as partes e os títulos com seus complementos;

·         Regulagem da linha: contém o detalhamento da regulagem da linha para esta versão de FT;

3.       Na pasta Dados da versão, os seguintes campos são exibidos, devendo ser preenchidos quando necessário:

                                                              i.      Operação: indica qual a operação que está sendo realizada (inclusão ou alteração);

                                                            ii.      Versão: número seqüencial da versão. Gerado automaticamente pelo sistema;

                                                          iii.      Nome da versão: deve ser preenchido o nome da versão da FT;

                                                           iv.      Aprovada: indica se a versão está aprovada ou não. Uma vez que a versão foi aprovada, não poderá voltar ao status não aprovada;

                                                             v.      Ativa: indica se a versão está ativa ou não. Somente poderá ter uma versão ativa para cada status de aprovação. Quando for emitida uma OP, a mesma será sempre gerada a partir da versão ativa;

                                                           vi.      Versão de custo: indica se está versão é a versão de custo. Uma FT somente poderá ter uma versão de custo;

                                                         vii.      Custo da matéria prima: campo calculado com base nas matérias primas informadas na pasta Partes da versão;

                                                       viii.      Custo operacional: campo onde será informado o custo operacional da versão da FT;

                                                          ix.      Observações internas: campo tipo texto onde serão informadas observações sobre  a versão da FT. Estas observações serão de uso interno, ou seja, não serão impressas na OP;

                                                            x.      Observações OP: campo tipo texto onde serão informadas observações sobre a versão da FT que serão impressas na OP.

4.       A pasta Partes da versão, é composta pelos seguintes agrupamentos de dados (grids):

·         Partes: carregadas automaticamente quando uma FT é incluída, com base nas partes do tipo de FT. Procedimentos para edição:

o        Para inserir ou alterar os dados de uma parte, deve-se dar um duplo clique sobre o nome da parte;

o        Na janela apresentada são solicitados os seguintes atributos:

§         Parte: apresenta o nome da parte;

§         Composição: se for uma FT composta, solicita a FT relacionada a esta parte;

§         Imprime: se esta parte deve ser impressa em uma OP.

·         Títulos: contém os títulos da parte. Para se visualizar os títulos de uma parte, deve-se selecionar uma parte. Procedimentos para edição:

o        No início do cadastramento de uma FT, esta lista estará vazia. Para se inserir novos registros, devem ser utilizados os botões logo abaixo do agrupamento (/ e O), para, respectivamente, incluir e excluir títulos para a parte selecionada;

o        Ao se clicar no botão para inclusão, será apresentada uma janela com todos os títulos vinculados a parte selecionada. O título a ser inserido deve ser selecionado através de um duplo clique sobre o mesmo;

o        Neste momento, o novo título foi inserido. Para ser complementado ou alterado, deve-se clicar sobre o mesmo. Neste momento, será aberta uma janela, apresentando os seguintes atributos:

§          Complemento: digitar o complemento do titulo, conforme tipo do titulo;

§         Rendimento: digitar o rendimento do material;

§         Rendimento de custo

·         Títulos dependentes: contém os títulos dependentes de um titulo do grupamento à esquerda. Para se visualizar os títulos dependentes, caso tenha, deve-se selecionar um titulo.

Tabelas Auxiliares

 

As tabelas auxiliares são tabelas que não estão envolvidas diretamente com os processos principais, porém são utilizadas pelos mesmos a fim de se manter uma organização nas informações. As operações disponíveis nas tabelas auxiliares são as seguintes:

-          Incluir à inclui um registro novo na tabela

-          Alterar à altera um registro selecionado

-          Excluir à exclui o registro selecionado

-          Cancelar à cancela a operação corrente

-          Imprimir à imprime os registros da tabela

-          Sair à sai da tabela

-          Pesquisar à busca por um registro na tabela (deve ser preenchido o campo indicando o que se pretende pesquisar)

-          Gravar à algumas operações necessitam que se grave as alterações, e nestes casos, a opção incluir se transforma em gravar

Os campos que se encontram em negrito são de preenchimento obrigatório, os demais são opcionais.

Partes

 

Agrupamento de informações que permitem a impressão de uma OP. Uma Ficha Técnica pode ser composta por várias partes, sendo que podem ser emitidas OP específicas para cada parte da Ficha Técnica. Os campos que devem ser preenchidos na tabela de Partes são os seguintes:

  • Nome da Parte: informar um nome significativo que permita identificar a parte;
  • Parte Fórmula: indicar se esta parte refere-se a uma fórmula de uma ficha técnica;

Neste momento, podem ser vinculados os títulos que pertencem a esta parte. Os títulos são vinculados utilizando-se os dois botões existentes na borda esquerda da janela. Provavelmente ainda não há títulos cadastrados no inicio do uso do sistema. Porém, os mesmos podem ser vinculados às partes na tabela de títulos ou posteriormente na opção alteração da tabela de partes.

Títulos

 

Os títulos são características das partes da ficha técnica, necessárias para descrever a parte ou detalhar uma fórmula. Os campos que devem ser preenchidos na tabela de títulos são os seguintes:

  • Nome do titulo: nome que será solicitado quando do preenchimento da ficha técnica. O nome do titulo será impresso no OP juntamente com o complemento somente se a parte não for formula;
  • Obrigatório: indica se o preenchimento do titulo é ou não obrigatório;
  • Tipo de complemento: indica qual o tipo de complemento que será solicitado quando o mesmo for preenchido. Os valores disponíveis são os seguintes: Caracter (o preenchimento será um texto livre), Número (o preenchimento será um valor numérico) ou Tabela (o preenchimento será um registro que virá de uma tabela do sistema);
  • Domínio: se o tipo de complemento for do tipo caracter ou número, poderá ser especificado um conjunto finito de complementos, que serão informados um em cada linha do campo domínio;
  • Prefixo: texto pré-determinado que antecede o complemento;
  • Sufixo: texto pré-determinado que sucede o complemento;
  • Máscara: especifica o formato da impressão dos complementos numéricos;
  • Título original: indica que este título está vinculado um outro título desta parte, ou seja, o seu preenchimento somente será requisitado se o título ao qual ele está vinculado tiver sido preenchido;
  • Tem rendimento: indica se o titulo compõe uma fórmula;
  • Imprime nome do título: indica se o nome do título será impresso na OP.

Neste momento, podem ser relacionadas as partes as quais os títulos serão vinculados, através dos dois botões que se encontram junto a borda esquerda da janela.

Tipos de Ficha Técnica

 

Tipos de Fichas Técnicas são categorizações de estruturas pré-definidas de Fichas Técnicas. Devem existir para que o processo de registro de Fichas Técnicas fique facilitado no sistema. Os campos que devem ser preenchidos são os seguintes:

  • Nome do Tipo FT: nome do tipo de ficha técnica;
  • Classificação: indica se este tipo de ficha técnica é composta por outros tipos de ficha técnica, ou não;
  • Código da família: indica qual a família de produtos que este tipo de ficha técnica está vinculado;
  • Tem grade: indica se este tipo de ficha técnica tem grade de numeração vinculada ou não;
  • Vira matéria prima: indica se as OP deste tipo de ficha técnica viram matéria prima quando são encerradas;
  • Imprime OP por parte: indica se devem ser impressas uma OP para cada parte da ficha técnica ou apenas uma OP.

Nesta mesma tela, ainda são apresentadas duas pastas. A primeira se refere as partes que compõem este tipo de ficha técnica. As partes podem ser incluídas ou excluídas, utilizando-se os dois primeiros botões que se encontram junto à margem esquerda. Os dois botões de baixo servem para que se mantenham ordenadas as partes que compõem a ficha técnica, quando impressas. A segunda pasta refere-se a composição. Ou seja, se o tipo de ficha técnica for composta, devem ser indicadas quais os tipos de ficha técnica que devem ser requisitadas quando uma ficha técnica deste tipo for inserida no sistema.

Tipos de Recurso

 

Tipos de recurso são agrupamentos de recursos existentes na produção, que podem ser considerados como tendo uma mesma função. Exemplo: misturador, extrusora, silo, etc. Os campos que devem ser preenchidos são os seguintes:

  • Nome do tipo de recurso: indica qual o nome do tipo de recurso;
  • Identificação do tipo de recurso: é uma especificação do tipo de capacidade que está vinculada a este tipo de recurso. Os tipos de capacidade são os seguintes:
    • Por capacidade OP  se o tipo de recurso é um limitante da capacidade total da OP;
    • Por capacidade de batelada  se o tipo de recurso é um limitante da batelada de uma OP;

 

    • Por capacidade hora  indica se o recurso é um limitante da capacidade por hora de produção;

Linhas de produção

 

Linhas de produção são conjuntos de recursos agrupados que implicam que os mesmos sejam alocados conjuntamente quando da emissão de uma OP. Os campos a serem preenchidos são os seguintes:

  • Nome da linha: indica qual o nome que será dado a cada linha existente;

Nesta mesma tabela, deve ser indicado, através do botão Regulagem, qual a regulagem dos recursos que pertencem a esta linha. Note que a regulagem será criada em no programa MS Word, sendo possível a utilização de todos os recursos existentes. O arquivo deverá ser salvo para que as alterações sejam gravadas.

Ainda deverão ser especificados os recursos pertencentes a esta linha, bem como a ordem que os mesmo se encontram na produção, através dos botões que se encontram junto à margem esquerda.

Matéria Prima de Laboratório

 

As matérias primas de laboratório são codificações de matérias primas internas a FCC. São especificadas por P&D e são relacionadas em Fichas Técnicas e OP. Uma mesma matéria prima de laboratório pode estar relacionada a mais de uma matéria prima, desde que elas tenham as mesmas propriedades e possam ser substituídas sem prejuízo a uma formulação. Nestes casos, diz-se que as matérias primas são contratipos. Os campos necessários para o registro de uma matéria prima de laboratório são os seguintes:

·         Código do material: digitar o código que a matéria prima de laboratório terá;

·         Nome do material: digitar o nome que identifique a matéria prima.

Na tela do cadastro das matérias primas de laboratório, ainda é necessário efetuar o vinculo das matérias primas de laboratório com as matérias primas do almoxarifado. Uma matéria prima somente pode estar vinculada a uma matéria prima de laboratório, e isto faz com que uma matéria prima, uma vez vinculada, não aparece mais na relação das matérias primas a vincular. O vinculo é feito através da utilização dos botões que ficam junto à margem esquerda da janela. Ainda, na mesma tela, deve ser feito o vinculo das matérias primas de laboratório com os títulos as quais estas matérias primas podem estar vinculadas. Caso não seja feito este vínculo, esta matéria prima de laboratório não aparecerá na relação de matérias primas disponíveis para composição de ficha técnica. Por exemplo, para um titulo com nome BORRACHA, devem ser vinculadas todas as matérias primas de laboratório que são borrachas.

Unidades de Consumo

 

As unidades de consumo são descrições da unidade das matérias primas de laboratórios que compõem as formulações de ficha técnica e conseqüentemente as OP. Se determinada matéria prima é utilizada na produção em sacos, então diz-se que a unidade de consumo é saco. Os campos necessários para o registro de uma unidade de consumo são os seguintes:

·         Nome da unidade de consumo: digitar o nome da unidade de consumo;

·         Medida: digitar sim caso a unidade de consumo utilize as medidas padrão para compor as formulações;

·         Quebra: digitar sim caso se queira controlar a quebra de uma unidade de consumo, e que o sistema gerencie o acumulado desta quebra.

Versão de FT

 

Toda e qualquer FT nasce com uma versão original. As alterações em sua estrutura (matérias primas, processos e recursos, com seus respectivos atributos) podem gerar uma nova versão da FT, a fim de permitir a evolução e os motivos que levaram ao mesmo. Somente uma versão poderá estar ativa e quanto a situação de aprovação, uma versão pode ser aprovada ou experimental.

Versão de Custo

 

Toda FT pode ter mais de uma versão, sendo que, uma delas deve obrigatoriamente ser definida como a versão de custo. A versão de custo é a versão que a controladoria definiu como sendo a base para o custeio das matérias primas da unidade de venda dos produtos a ela associados. No momento da emissão de uma OP, o custo de produção sempre será comparado ao custo da versão de custo, e caso ultrapasse, um usuário com poderes para liberar a emissão da OP devera ser comunicado.
